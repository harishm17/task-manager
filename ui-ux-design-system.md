## Executive Summary

This app faces a **critical UX challenge**: Managing TWO distinct data types (tasks + expenses) across MULTIPLE groups without overwhelming users. Poor design = confused users = churn.

### Core UX Principles

1. **Progressive Disclosure** - Show features as needed, not all at once
2. **Context Awareness** - UI adapts to current group (personal vs household)
3. **Visual Hierarchy** - Most important actions are most prominent
4. **Consistent Patterns** - Similar actions look and behave similarly
5. **Clear Feedback** - Every action has immediate visual confirmation
6. **Mobile-First** - Design for small screens, enhance for desktop

### Key Design Decisions

âœ… **Navigation:** Bottom tabs (mobile) + Sidebar (desktop)
âœ… **Group Switching:** Dropdown in top nav (always visible)
âœ… **Tasks vs Expenses:** Separate tabs, visually distinct
âœ… **Balances:** Dedicated "Balances" tab, not buried in expenses
âœ… **Color System:** Semantic colors (green = owed, red = owe, blue = tasks)
âœ… **Primary Action:** Floating Action Button (FAB) on mobile, top-right button on desktop

## 11. Accessibility (A11y) Considerations

### 11.1 Keyboard Navigation

**All interactive elements must be keyboard-accessible:**
- Tab: Move to next element
- Shift+Tab: Move to previous element
- Enter/Space: Activate button/checkbox
- Esc: Close modal/dropdown
- Arrow keys: Navigate within dropdowns/lists

**Focus indicators:**
- Blue outline (2px solid) around focused element
- Must be visible on all backgrounds

### 11.2 Screen Reader Support

**ARIA labels for icons:**
```html
<button aria-label="Add new task">
  <PlusIcon />
</button>
```

**Status announcements:**
```html
<div role="status" aria-live="polite">
  Task "Clean bathroom" marked as completed
</div>
```

### 11.3 Color Contrast

**WCAG AA compliance (minimum):**
- Text on background: 4.5:1 contrast ratio
- Large text (18px+): 3:1 contrast ratio
- UI elements (buttons, borders): 3:1 contrast ratio

**Don't rely on color alone:**
- âœ… "You owe $50" (red text) + "â–² You owe" icon
- âŒ Just red text with no label

## 15. Summary: Key UX Principles to Follow

### 1. **Progressive Disclosure**
- Don't show everything at once
- Reveal features as user needs them
- Example: Hide "Expenses" tab in personal groups

### 2. **Context Awareness**
- UI adapts to current group (personal vs household)
- Always show which group user is viewing
- FAB shows "Add Task" or "Add Expense" based on context

### 3. **Clear Hierarchy**
- Most important info at top (balances, due tasks)
- Primary actions (Add, Settle Up) most prominent
- Secondary actions (View Details) less prominent

### 4. **Consistent Patterns**
- Similar actions look similar (Add Task, Add Expense forms)
- Same colors mean same thing everywhere (red = owe)
- Same gestures work everywhere (swipe to complete)

### 5. **Immediate Feedback**
- Every action has visual confirmation (toast, animation)
- No silent failures (always show errors)
- Optimistic updates (feel instant)

### 6. **Mobile-First Thinking**
- Design for small screens first
- Large touch targets (44px minimum)
- Thumb-friendly navigation (bottom tabs, FAB)

### 7. **Clarity Over Cleverness**
- Use simple language ("You owe $50" not "Debt: $50")
- Show calculations ("Each person owes $37.50")
- Explain what's happening ("Payment recorded; balances updated")
